Key Concepts

Seven essential terms and ideas to understand HTTP.

1. HTTP (Hypertext Transfer Protocol)

The application-layer protocol used to transfer data on the web. It defines how requests from clients (browsers) and responses from servers are structured and communicated. Every webpage load relies on HTTP.

2. Request / Response Cycle

The fundamental pattern of HTTP communication. The client sends a request with a method, URL, and headers. The server sends back a response with a status code, headers, and (usually) a body containing data.

3. Stateless Protocol

HTTP is stateless, meaning each request is independent. The server does not remember previous requests from the same client. Cookies and sessions are used to maintain state across multiple requests.

4. HTTPS & TLS

HTTPS (HTTP Secure) adds a layer of encryption using Transport Layer Security (TLS). It encrypts data in transit and verifies server identity using certificates issued by trusted Certificate Authorities (CAs).

5. Persistent Connections

Introduced in HTTP/1.1, persistent connections allow multiple requests to reuse a single TCP connection. This reduces overhead from repeatedly opening and closing connections and improves page load times.

6. Multiplexing

A key feature of HTTP/2 that allows multiple requests and responses to travel simultaneously over a single connection without blocking each other. This significantly improves performance on modern web pages with many resources.

7. QUIC Transport Protocol

QUIC is the transport protocol underlying HTTP/3, developed by Google and standardized by the IETF. Unlike TCP, QUIC was designed with modern web use cases in mind, including faster connection setup (0-RTT in some cases), built-in encryption, and better handling of packet loss on mobile and unstable networks.

Quick Comparison: HTTP vs HTTPS

Feature HTTP HTTPS
Encryption None TLS encryption
Data privacy Data visible to intermediaries Data is encrypted in transit
Authentication None Server identity verified via certificates
Browser indicator Warning displayed Padlock icon shown
Required for Basic sites (discouraged) Login, payments, modern features
